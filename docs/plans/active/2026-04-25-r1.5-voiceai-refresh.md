## Phases

### Phase 1 — Provider implementations (~4-5 días, parallel)

Three independent tracks, parallelizable with subagent-driven batching. Tracks are fully isolated from each other (different folders, different test files), so subagents do not collide.

#### Track 1.A — ElevenLabs Flash 2.5 extension (1 día, S)

Foreground or background subagent OK; small surface, low risk.

- T1.A.1 — Add [`ElevenLabsModels`](../../../src/Asterisk.Sdk.VoiceAi.Tts/ElevenLabs/) constants class (`Flash25 = "eleven_flash_v2_5"`, `Turbo2`, `Multilingual2`). Verify model id at impl time against [ElevenLabs models docs](https://elevenlabs.io/docs/overview/models).
- T1.A.2 — Extend `ElevenLabsOptions` with additive properties: `LatencyOptimization` enum (Off…Max, 0-4 scale per provider's `optimize_streaming_latency`), `OutputFormat` enum (Pcm16k / Pcm22050 / Pcm24k), `Model` default flip to `Flash25`.
- T1.A.3 — Update `ElevenLabsSpeechSynthesizer` WS connection to surface model + format + latency-optim parameters per provider docs.
- T1.A.4 — Unit tests: options binding, WS request shape verification (using mock `IClientWebSocket`), Flash 2.5 default behavior.
- T1.A.5 — Integration test (gated `RUN_LIVE_TESTS=true` + `ELEVENLABS_API_KEY` env): assert TTFA < 200 ms ceiling.
- T1.A.6 — `PublicAPI.Unshipped.txt` updated with new types.

#### Track 1.B — Deepgram Aura 2 TTS (new provider, **WebSocket** endpoint) (2 días, M)

Background subagent recommended. Mirror existing Deepgram STT structure for shape consistency.

- T1.B.0 — **Spec Q1 probe** (~½ día) — write a 50-line probe that connects to `wss://api.deepgram.com/v1/speak?model=aura-2-thalia-en`, sends one phrase via JSON `{"type":"Speak","text":"hello"}`, prints frame metadata (sizes, types, content-types, status events). Goal: verify the documented frame protocol matches reality. Probe lives in `Examples/` or as a one-off test fixture; deleted after T1.B.6.
- T1.B.1 — Create folder `src/Asterisk.Sdk.VoiceAi.Tts/Deepgram/` with options + synthesizer + DI extension + JSON serialization context entries.
- T1.B.2 — Implement `DeepgramSpeechSynthesizer : SpeechSynthesizer` with the **WebSocket** endpoint (`wss://api.deepgram.com/v1/speak?...`); auth `Authorization: Token <key>`; streaming PCM 16-bit / mu-law / a-law per `DeepgramTtsOptions.Encoding`. **Do NOT implement REST `/v1/speak` fallback in v1.15.3** — WS only.
- T1.B.3 — Add voice catalog constants `DeepgramVoices.Thalia`, `Andromeda`, etc. (per Aura 2 voice list, including 2026 multilingual additions).
- T1.B.4 — Health check: pings the Deepgram TTS lightweight endpoint on startup; degrades if 401/503.
- T1.B.5 — Unit tests: 8-10 covering options binding, WS request shape, frame yield correctness, error handling on disconnect.
- T1.B.6 — Integration test (gated, `DEEPGRAM_API_KEY` env): asserts TTFA < 250 ms ceiling + sample ≥ 1 second of audio frames yielded.
- T1.B.7 — `PublicAPI.Unshipped.txt` updated.

### Phase 2 — TTFA observability + cross-cutting (1 día)

Foreground (depends on Phase 1 completion — TTFA recording is base-class behavior, applies to all providers including the new ones).

- T2.1 — Add `tts.synthesis.ttfa_ms` `Histogram<double>` to [`SpeechSynthesisMetrics`](../../../src/Asterisk.Sdk.VoiceAi/Diagnostics/SpeechSynthesisMetrics.cs) with description "Time-to-first-audio: elapsed from synthesis start until first audio frame yielded to caller." Tags: `voiceai.provider`, `tts.model`.
- T2.2 — Modify `SpeechSynthesizer` base class to wrap `SynthesizeAsync` and record TTFA on first yielded byte. Use a `Stopwatch` started at method entry; record once on first frame.

For any provider that fails: file an issue, fix or document the buffer-then-yield pattern.
- T2.4 — Update [`src/Asterisk.Sdk.VoiceAi.Tts/README.md`](../../../src/Asterisk.Sdk.VoiceAi.Tts/README.md) listing the new histogram with default histogram buckets per Asterisk OTel conventions (likely 5/10/25/50/100/250/500/1000/2500/5000 ms).
- T2.5 — Unit tests: 4-5 covering TTFA recording correctness (records on first yield, not on subsequent yields, not on completion).
- T2.6 — Bench test: BenchmarkDotNet smoke in `Tests/Asterisk.Sdk.Benchmarks/` exercising TTFA path (env-gated, IT-style).

## Subagent-Driven execution rules

- **Q1 probe (T1.B.0) and Q2 verification (T1.C.0):** foreground, sequenced before each track's implementation. Cheap (~½ day each); answer is required before committing 1-2 days of impl per track.
- **Phase 1 Tracks A + B + C (background, parallel):** launch 3 subagents in single message after Q1/Q2 resolved, monitor with completion notifications. Each track is fully isolated — different folders, different tests, no shared state.
- **Phase 2 (foreground):** depends on Phase 1; sequential after parallel tracks merge.
- **Phase 3 (foreground):** release engineering requires user input on tag/release timing.

**Per-subagent prompt template:**
- "You are implementing R1.5 Track 1.X. Spec: `[path]`. Existing reference impl: `[path to most-similar provider — Cartesia for LMNT, Deepgram STT for Aura 2 TTS, ElevenLabs current for Flash 2.5]`. Strict requirements: zero AOT warnings, AOT-safe DI (no reflection), JSON serializer context registered, health check auto-registered, unit tests covering happy path + 2 edge cases per public method, `PublicAPI.Unshipped.txt` updated."
- Each subagent reports: files created/modified, test count delta, AOT publish output, any deviations from spec with rationale.

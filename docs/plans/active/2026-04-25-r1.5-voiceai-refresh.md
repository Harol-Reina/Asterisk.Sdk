### Phase 1 — Provider implementations (~4 días, parallel)

Three independent tracks, parallelizable with subagent-driven batching.

**Track 1.A — ElevenLabs Flash 2.5 extension** (1 día, S)

Foreground or background subagent OK; small surface, low risk.

- T1.A.1 — Add `ElevenLabsModels` constants class with `Flash25`, `Turbo2`, `Multilingual2`.
- T1.A.2 — Extend `ElevenLabsOptions` with `LatencyOptimization` enum + `OutputFormat` enum + `Model` default flip to `Flash25`.
- T1.A.3 — Update `ElevenLabsSpeechSynthesizer` WS connection to surface model + format + latency-optim parameters per provider docs.
- T1.A.4 — Unit tests: options binding, WS request shape verification (using mock `IClientWebSocket`).
- T1.A.5 — Integration test: gated `RUN_LIVE_TESTS=true` + `ELEVENLABS_API_KEY` env, assert TTFA < 200ms ceiling.

**Track 1.B — Deepgram Aura 2 TTS new provider** (2 días, M)

Background subagent recommended. Mirror existing Deepgram STT structure for shape consistency.

- T1.B.1 — Create folder `Asterisk.Sdk.VoiceAi.Tts/Providers/Deepgram/` with options + synthesizer + DI extension + JSON serialization context entries (or new context).
- T1.B.2 — Implement `DeepgramSpeechSynthesizer : SpeechSynthesizer` with Deepgram `/v1/speak` WS endpoint; auth `Authorization: Token <key>`; streaming PCM 16-bit mono frames.
- T1.B.3 — Add voice catalog constants `DeepgramVoices.Thalia`, `Andromeda`, etc. (per Aura 2 voice list).
- T1.B.4 — Health check: pings `/v1/projects` (lightweight Deepgram health endpoint) on startup; degrades if 401/503.
- T1.B.5 — Unit tests: 8-10 covering options binding, WS request, frame yield correctness.
- T1.B.6 — Integration test: gated, asserts TTFA < 250ms ceiling + sample ≥ 1 second of audio frames yielded.

**Track 1.C — Whisper V3 local STT new provider** (3-4 días, M-L; depends on Phase 0 outcome)

Background subagent. Higher risk track; isolate from 1.A/1.B to keep paralelization clean.

- T1.C.1 — Create folder `Asterisk.Sdk.VoiceAi.Stt/Providers/WhisperLocal/` with options + recognizer + DI extension.
- T1.C.2 — Implement `WhisperLocalSpeechRecognizer : SpeechRecognizer` per Phase 0 outcome (Whisper.net or direct P/Invoke).
- T1.C.3 — Internal upsampler 8 kHz → 16 kHz (linear interp) + frame buffering + VAD-based segment delimiting.
- T1.C.4 — Health check: verifies model file exists + load probe at startup.
- T1.C.5 — Documentation: `docs/voiceai/whisper-local-setup.md` walks through model download from Hugging Face + setup script `scripts/download-whisper-model.sh`.
- T1.C.6 — Unit tests: 6-8 covering options validation (path missing → fail-fast), upsampler correctness, segment buffering.
- T1.C.7 — Integration test: gated, downloads `tiny.bin` (~75 MB) on first CI run, transcribes 3 fixed WAV samples, asserts transcripts match ground truth.

## Subagent-Driven execution rules

- **Phase 0 (foreground):** AOT spike requires real-time iteration on warnings. Foreground subagent acceptable.
- **Phase 1 Tracks A + B + C (background, parallel):** launch 3 subagents in single message, monitor with completion notifications. Each track is fully isolated from the others — different folders, different test sets.
- **Phase 2 (foreground):** depends on Phase 1; sequential after parallel tracks merge.
- **Phase 3 (foreground):** release engineering requires user input on tag/release timing.

**Per-subagent prompt template:**
- "You are implementing R1.5 Track 1.X. Spec: `[path]`. Existing reference impl: `[path to most-similar provider]`. Strict requirements: zero AOT warnings, AOT-safe DI, JSON serializer context registered, health check auto-registered, unit tests covering happy path + 2 edge cases per public method."
- Each subagent reports: files created/modified, test count delta, AOT publish output, any deviations from spec with rationale.

**Date:** 2026-04-25
**Status:** Approved (D-FORCE-4 from Platform post-ship R5.1 triage)

**Branch:** `r1.5-voiceai-refresh`

## Context

R5 release train identifies R1.5 as a paralelo oportunista — pure SDK-side voice-AI provider refresh, zero deps with R5 admin/security work. Ship target: any time during the QA gap windows of R5.1→R5.2 or R5.2→R5.3. Branch is intentionally separate from R5 cadence.

**Three concrete deliverables** carried forward from R5 spec:

1. **ElevenLabs Flash 2.5** TTS — newer model on the existing `Asterisk.Sdk.VoiceAi.Tts.ElevenLabs` provider. Target `<150ms TTFA` (time-to-first-audio) competitive vs Cartesia Sonic and OpenAI gpt-4o-tts.
2. **Deepgram Aura 2** TTS — **new provider** in `Asterisk.Sdk.VoiceAi.Tts/Providers/Deepgram/` (Deepgram today is STT-only via Nova-2; Aura 2 is their new TTS model). Adds another low-latency option in the TTS catalog.
3. **Whisper V3 local** STT — **new provider** under `Asterisk.Sdk.VoiceAi.Stt/Providers/WhisperLocal/` using `Whisper.net` (P/Invoke wrapper over whisper.cpp). Air-gapped; no internet egress required. First "local / embedded" provider in the SDK; sets a precedent for future on-prem privacy-sensitive deploys.

Plus one **observability gap** flagged by the SDK audit: **TTFA histogram is not exported today**. The TTS meter `Asterisk.Sdk.VoiceAi.Tts` exposes a generic latency histogram but does not differentiate first-audio time from total-synthesis time. Flash 2.5's marketing claim is exactly TTFA, so the metric must exist before the provider lands or the claim is unverifiable.

Plus one **Pro consumer wiring**: `Asterisk.Sdk.Pro.AgentAssist.AgentAssistBuilder` exposes fluent `WithDeepgramStt()`, `WithWhisperStt()`, `WithAzureWhisperStt()`, `WithGoogleStt()`, `WithElevenLabsTts()`, `WithAzureTts()` extensions today. R1.5 adds `WithDeepgramTts()` and `WithWhisperLocalStt()`. ElevenLabs Flash 2.5 is a model-level change reachable via existing options (no new builder method needed).

## Out of scope

- **Pro / Platform / Web changes.** Pro consumes new providers via auto-detection (when AgentAssist runtime feature toggle UI inspects available providers, new entries appear in the dropdown). Pro is bumped to consume v1.15.3 only when there's reason to (next R5 release).
- **Cartesia Sonic-3 refresh** — already current as of v1.15.0; not part of R1.5.
- **OpenAI Realtime API integration** — separate paquete `Asterisk.Sdk.VoiceAi.OpenAiRealtime`, not touched in R1.5.
- **AudioSocket frame-format changes** — out of scope; existing `Slin16Mono8kHz` / `Slin16Mono16kHz` formats are sufficient.
- **Custom voice cloning / voice library management** — ElevenLabs feature surface beyond Flash 2.5 model selection.

## Open technical questions (resolved by Phase 0 spike before full impl)

### Q1 — Whisper.net AOT compatibility

`Whisper.net` is a P/Invoke wrapper over `whisper.cpp` (native C/C++). Native interop generally works under AOT but has known footguns:

- Trim warnings on reflection-based marshaling.
- `LibraryImport` source generators (vs old `DllImport`) required for AOT-clean P/Invoke.
- Native binary loading per-RID (Linux x64, macOS arm64, Windows x64) — handled by NuGet's runtime-graph but must be validated.

**Decision rule:** Phase 0 spike (1-2 days) — write a minimal `WhisperLocalSpeechRecognizer` that loads `Whisper.net`, transcribes 5 seconds of audio, runs under `dotnet publish -p:PublishAot=true` and `dotnet publish -p:PublishTrimmed=true`. Two outcomes:

- **(a) Clean publish (zero IL3050/IL3053 warnings):** proceed with full implementation. Whisper.net is the chosen library.
- **(b) Warnings present but suppressible with attribute-narrow `UnconditionalSuppressMessage`:** proceed with documented suppressions justified by audit.
- **(c) Warnings unsuppressible / native loading broken under AOT:** fall back to direct P/Invoke against `whisper.cpp` with `LibraryImport`-source-generated bindings. Higher cost (2-3 extra days) but full AOT control.

Phase 0 outputs a **go/no-go decision** before Phase 1 starts. Spike code is throwaway; final impl follows the chosen path.

### Q2 — Whisper V3 model file packaging

Whisper V3 model files are 1-3 GB depending on size variant (tiny / base / small / medium / large / large-v3). Cannot bundle in NuGet.

**Decision:** **Operator-provided model path**, no auto-download from network. Reasons:

- Air-gapped is the explicit value prop. Auto-download contradicts it.
- Model file licensing varies; operators must accept their license model deliberately.
- Default options expose `WhisperLocalOptions.ModelPath = "/var/lib/asterisk-sdk/whisper/ggml-large-v3.bin"`; provider fails fast at startup if file missing or model load fails.
- Documentation in `docs/voiceai/whisper-local-setup.md` walks through model download from `huggingface.co/ggerganov/whisper.cpp` (compatible models) plus a `scripts/download-whisper-model.sh` helper script (operator runs explicitly).

**Alternative considered + rejected:** auto-download with checksum verification on first init. Rejected — adds startup latency, breaks air-gapped narrative, requires SDK to choose a CDN/registry.

### Q3 — Audio format conversion

Whisper expects 16-bit PCM mono **16 kHz**. SDK pipeline default is `Slin16Mono8kHz`. Two paths:

- **(a) Resample inside the provider:** provider does linear-interp upsample 8 → 16 kHz internally. Simple but unnecessarily duplicates work if other providers also need 16 kHz.
- **(b) Resample upstream in shared pipeline:** add `AudioFormat.Slin16Mono16kHz` flow to AudioSocket / pipeline orchestrator with explicit format negotiation per provider.

**Decision:** **(a) Resample inside provider** for R1.5 scope. Simple, isolated, no spillover into AudioSocket / pipeline. Future R2/v2-preview1 can introduce shared resampler abstraction if 3+ providers end up needing the same conversion. WhisperLocal documents the internal upsample in XML doc + spec.

## Design

### Track 1 — ElevenLabs Flash 2.5 (existing provider extension)

`Asterisk.Sdk.VoiceAi.Tts/Providers/ElevenLabs/ElevenLabsOptions.cs` already exposes a `Model` property (today defaults to `eleven_turbo_v2`). Changes:

- Add canonical constants: `ElevenLabsModels.Flash25 = "flash_v2_5"`, `Turbo2 = "eleven_turbo_v2"`, `Multilingual2 = "eleven_multilingual_v2"`.
- Update default to `Flash25` (breaking? — re-verified: existing options accept any string; constant is documentation, default flip is safe additive).
- Add `OutputFormat` option (PCM 16-bit @ 16 kHz / 22.05 kHz / 24 kHz; today fixed via internal mapping). Required for Flash 2.5's lower-latency PCM frames.
- Add `LatencyOptimization` enum mapping to the provider's `optimize_streaming_latency` URL parameter (0-4 scale per their API).
- Update `ElevenLabsSpeechSynthesizer` WebSocket request to surface the new model + format + latency optimization.

**Backward compat:** existing options default behavior preserved if `Model` not set; new code paths only trigger when Flash 2.5 selected.

### Track 2 — Deepgram Aura 2 TTS (new provider)

New folder `Asterisk.Sdk.VoiceAi.Tts/Providers/Deepgram/`:

- `DeepgramTtsOptions.cs` — ApiKey, Model (default `aura-2-thalia-en`; constants for available voices), SampleRate, Encoding, Container.
- `DeepgramSpeechSynthesizer.cs` — WebSocket streaming via Deepgram's `/v1/speak` real-time endpoint, returning PCM 16-bit mono. Implements `SpeechSynthesizer` base.
- `DependencyInjection/DeepgramTtsServiceCollectionExtensions.cs` — `AddDeepgramSpeechSynthesizer(opts => ...)` + auto-registered `TtsHealthCheck` that pings the Deepgram TTS endpoint.
- JSON serializer context entries in `Internal/VoiceAiTtsJsonContext.cs` (or per-provider JSON context if pattern dictates — match existing Deepgram STT pattern).

Mirror Deepgram STT structure (already in repo as Nova-2 STT) for code-shape consistency. Same `HttpClient` setup pattern, same auth header (`Authorization: Token <key>`), same WebSocket exception handling.

### Track 3 — Whisper V3 local STT (new provider)

New folder `Asterisk.Sdk.VoiceAi.Stt/Providers/WhisperLocal/` (or new project `Asterisk.Sdk.VoiceAi.Stt.WhisperLocal` if Phase 0 reveals AOT-isolation benefits):

- `WhisperLocalOptions.cs` — ModelPath (required), Language (`auto` / `en` / `es` / `pt`), Threads (default `Environment.ProcessorCount / 2`), GpuDevice (`cpu` / `cuda` / `metal`; default `cpu` for air-gapped baseline), Translate (bool, secondary feature).
- `WhisperLocalSpeechRecognizer.cs` — implements `SpeechRecognizer` base. On `Initialize`: load model file via Whisper.net's `WhisperFactory.FromPath(path)`, build `WhisperProcessor` with options. On `RecognizeAsync(audioFrames)`: accumulate frames into a buffer, run inference on speech segments delimited by VAD (Whisper.net's built-in VAD or simple energy threshold).
- Internal upsampler 8 → 16 kHz (linear interp) as decided in Q3.
- `DependencyInjection/WhisperLocalServiceCollectionExtensions.cs` — `AddWhisperLocalSpeechRecognizer(opts => ...)` + `SttHealthCheck` that verifies model file exists + load probe.
- `JsonSerializerContext` only if Whisper.net response objects need to be serialized (likely only for diagnostics).
- Native binary support: NuGet runtime graph handled by Whisper.net package; documented per-RID coverage in `docs/voiceai/whisper-local-setup.md`.

**Latency target:** comparable to OpenAI Whisper REST (~1-3s per 30s audio) on CPU; faster with GPU. Not real-time streaming today (Whisper architecture is non-streaming); document as batch-on-final-segment in XML doc + spec note.

### Track 4 — TTFA observability gap closure

`Asterisk.Sdk.VoiceAi.Tts.Diagnostics.SpeechSynthesisMetrics`:

- Add histogram: `tts.synthesis.ttfa_ms` ("Time to first audio frame returned to caller, in milliseconds").
- Recording: `SpeechSynthesizer` base wraps `RecognizeAsync` → measure elapsed from method entry to first yielded `ReadOnlyMemory<byte>`.
- Tags: `voiceai.provider`, `tts.model`.
- Existing `tts.synthesis.latency_ms` (total) preserved unchanged.

Documentation update: `Asterisk.Sdk.VoiceAi.Tts/Diagnostics/README.md` (or equivalent) lists the new histogram with default histogram buckets per Asterisk OTel conventions.

### Track 5 — Pro consumer wiring (Pro side, deferred to Pro v1.13.x-pro patch)

Out of R1.5 SDK scope per "Out of scope" above, but documented here so Pro brainstorm has the bridge ready:

- `Asterisk.Sdk.Pro.AgentAssist.AgentAssistBuilder.WithDeepgramTts(Action<DeepgramTtsOptions>?)` — fluent builder method.
- `WithWhisperLocalStt(Action<WhisperLocalOptions>?)` — same.
- ElevenLabs Flash 2.5 is reachable today via `WithElevenLabsTts(opts => opts.Model = ElevenLabsModels.Flash25)`.

**Pro Track 5** lands in next Pro release alongside whichever R5 bump consumes SDK v1.15.3.

New types (all `public`):

```csharp
// SDK Asterisk.Sdk.VoiceAi.Tts.ElevenLabs:

public enum ElevenLabsLatencyOptimization { Off = 0, Low = 1, ..., Max = 4 }

// SDK Asterisk.Sdk.VoiceAi.Tts.Deepgram (new namespace):
public sealed class DeepgramTtsOptions { ... }
public sealed class DeepgramSpeechSynthesizer : SpeechSynthesizer { ... }

    public static IServiceCollection AddDeepgramSpeechSynthesizer(
        this IServiceCollection services, Action<DeepgramTtsOptions>? configure = null);
}

// SDK Asterisk.Sdk.VoiceAi.Stt.WhisperLocal (new namespace):

```

All types follow the existing provider layout and conventions. Zero breaking changes; v1.15.3 is a strictly additive minor patch.

## Test strategy

- **Unit tests:** mock-based against fakes (`FakeSpeechRecognizer`, `FakeSpeechSynthesizer` extended for new metrics). New ~30-40 tests across 3 providers + TTFA metric coverage.
- **Integration tests** (Testcontainers / live, `RUN_LIVE_TESTS=true` env):
  - ElevenLabs Flash 2.5: API key required, generates 5-second sample, asserts TTFA < 200ms (env-tolerant ceiling above the 150ms claim).
  - Deepgram Aura 2: API key required, similar TTFA assertion.
  - Whisper V3 local: model file path required (CI runner downloads `tiny.bin` only ~75 MB to stay under CI runtime budget), assert transcript matches ground-truth on 3-4 fixed sample files.
- **AOT publish smoke:** new CI job `r1-5-aot-validation` running `dotnet publish -c Release -p:PublishAot=true -r linux-x64 ./tests/Asterisk.Sdk.VoiceAi.AotSmoke/` and asserting zero IL3050/IL3053. Phase 0 spike output feeds into this job's expectations.
- **Latency benchmark:** `tests/Benchmark.VoiceAi/` BenchmarkDotNet smoke job — 10-run average TTFA per provider, gated by env var. Output appended to PR description for review.

## Risks / mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| Whisper.net AOT-incompatible | Medium | Phase 0 spike (Q1 above); fallback path (direct `LibraryImport` P/Invoke) costs +2-3 days but is contained. |
| Deepgram Aura 2 API still in preview / unstable | Low-Medium | Flag in code as "preview API, may evolve" XML doc; ship anyway to capture early adopters. Sales narrative: "early access to Aura 2". |
| Flash 2.5 TTFA claim unverifiable in test env | Low | TTFA histogram exists; env-tolerant ceiling in IT (200ms vs 150ms target). Marketing claim references provider documentation, not our test. |
| Whisper V3 model file licensing | Low | Operator-provided path forces explicit license acceptance; SDK doesn't ship or distribute model files. |
| Native binary missing on operator deploy | Low | Whisper.net NuGet runtime graph handles per-RID; fallback error message names the missing native lib + setup doc URL. |
| Dependency tree bloat | Low-Medium | Whisper.net adds ~1.5 MB of managed code + native binaries per RID (~30 MB Linux x64). Document size impact in CHANGELOG. |

## Context

R5 release train identifies R1.5 as a paralelo oportunista — pure SDK-side voice-AI provider refresh, zero deps with R5 admin/security work. Ship target: any time during the QA gap windows of R5.1→R5.2 or R5.2→R5.3. Branch is intentionally separate from R5 cadence.

**Three concrete deliverables in v2 scope:**

1. **ElevenLabs Flash 2.5** TTS — newer model on the existing `Asterisk.Sdk.VoiceAi.Tts/ElevenLabs/` provider. Target `<150ms TTFA` (time-to-first-audio) competitive vs Cartesia Sonic and OpenAI gpt-4o-tts. *(Note: Eleven v3, GA 2026-03-14, is intentionally NOT a candidate — v3 is the expressive flagship for non-realtime use; for telephony Flash 2.5 remains the correct target per ElevenLabs' own latency guidance.)*
2. **Deepgram Aura 2** TTS — **new provider** in `Asterisk.Sdk.VoiceAi.Tts/Deepgram/`. Integrates Deepgram's **WebSocket** TTS endpoint (`wss://api.deepgram.com/v1/speak?model=aura-2-thalia-en`) with token-by-token input streaming, NOT the older REST `/v1/speak`. The WS endpoint is ~70% faster LLM→TTS than REST per Deepgram's published benchmarks. (REST may be added later as a fallback.)
3. **LMNT TTS** — **new provider** in `Asterisk.Sdk.VoiceAi.Tts/Lmnt/`. LMNT is the sub-200 ms TTS for conversational AI (per 2026 third-party benchmarks). Its niche is exactly the SDK target market: low-latency voice agents. Adds genuine differentiation to the TTS catalog (vs the prior plan that just refreshed two existing names).

Plus one **observability gap** flagged by the SDK audit: **TTFA histogram is not exported today**. The TTS meter `Asterisk.Sdk.VoiceAi.Tts` (in `src/Asterisk.Sdk.VoiceAi/Diagnostics/SpeechSynthesisMetrics.cs`) exposes `tts.synthesis.latency_ms` (total) but does not differentiate first-audio time from total-synthesis time. Flash 2.5 / Aura 2 / LMNT all market themselves on TTFA, so the metric must exist before the providers land or the claims are unverifiable.

Plus one **Pro consumer wiring**: `Asterisk.Sdk.Pro.AgentAssist.AgentAssistBuilder` exposes fluent `WithDeepgramStt()`, `WithWhisperStt()`, `WithAzureWhisperStt()`, `WithGoogleStt()`, `WithElevenLabsTts()`, `WithAzureTts()` extensions today. R1.5 v2 adds `WithDeepgramTts()` and `WithLmntTts()`. ElevenLabs Flash 2.5 is a model-level change reachable via existing options (no new builder method needed).

## Out of scope

- **Whisper V3 local STT** — **deferred to a future `Asterisk.Sdk.VoiceAi.Stt.OnPremPrivacy` track** (no firm date). Reasons: (1) quality unfit for telephony — Whisper degrades WER ~30-40% on narrowband 8 kHz audio upsampled to 16 kHz; existing cloud STT options (Deepgram Nova-3, Speechmatics, AssemblyAI) outperform it on the SDK's actual telephony use case; (2) Whisper.net (P/Invoke wrapper) has no confirmed Native AOT support in any release notes — last stable was v1.9.0 (Nov 2024) with only "NativeLibs" pre-releases since; the AOT spike outcome is genuinely uncertain and could blow the schedule with outcome (c) +2-3 days for direct `LibraryImport` rewrites; (3) the real value prop for local Whisper is **air-gapped on-prem deployments**, which deserves a dedicated track with CMK encryption / model-license workflow / SBOM disclosures, not a corner of a TTS-focused refresh.
- **Pro / Platform / Web changes.** Pro consumes new providers via auto-detection (when AgentAssist runtime feature toggle UI inspects available providers, new entries appear in the dropdown). Pro is bumped to consume v1.15.3 only when there's reason to (next R5 release).
- **Cartesia Sonic-3 refresh** — already current as of v1.15.0; not part of R1.5.
- **OpenAI Realtime API integration** — separate paquete `Asterisk.Sdk.VoiceAi.OpenAiRealtime`, not touched in R1.5.
- **AudioSocket frame-format changes** — out of scope; existing `Slin16Mono8kHz` / `Slin16Mono16kHz` formats are sufficient.
- **Custom voice cloning / voice library management** — ElevenLabs feature surface beyond Flash 2.5 model selection.
- **Turn-detection / barge-in** (smart-turn-v3, semantic VAD) — identified as a high-value gap during the v2 audit, but out of R1.5 scope. Tracked separately as a candidate for `v1.16` "Conversation Primitives" or similar.

## Open technical questions (resolved before / during impl)

### Q1 — Deepgram Aura 2 WebSocket auth + frame protocol

The WS endpoint authenticates via `Authorization: Token <key>` (same as Deepgram STT) but the **frame protocol is distinct from STT**: client sends JSON `{ "type": "Speak", "text": "..." }` messages and receives binary PCM frames + JSON status events interleaved. Need to verify exact message schema before committing to a `DeepgramSpeechSynthesizer` implementation.

**Resolution:** ~½ day API exploration before Track 1.B starts. Deliverable: a 50-line probe in `Examples/` (or a unit-test fixture) that authenticates, sends one phrase, and prints frame metadata. No code commitment until the probe runs cleanly.

### Q2 — LMNT API contract: SDK package vs raw HTTP

LMNT publishes a Python SDK + a REST/WebSocket API. There's no first-party .NET SDK as of May 2026.

**Decision:** **Raw HTTP/WebSocket, no third-party wrapper.** Rationale: (a) the SDK already follows this pattern for VoiceAi providers (per ADR-0014 — raw HTTP for VoiceAi to keep AOT-clean); (b) avoids depending on a Python-translation NuGet of unknown maintenance; (c) keeps the public API surface coherent with existing providers. Implementation uses `HttpClient` + `IClientWebSocket` from the existing internal abstractions.

### Q3 — Audio format conversion for input streams

Both Aura 2 WebSocket and LMNT accept input *text* token-by-token, NOT audio. So Q3 from v1 (audio resampling for Whisper) is **no longer relevant**. The previous concern about `Slin16Mono8kHz` → `Slin16Mono16kHz` upsampling does not apply to TTS providers. **Removed.**

## Design

### Track 1.A — ElevenLabs Flash 2.5 (existing provider extension)

`Asterisk.Sdk.VoiceAi.Tts/ElevenLabs/ElevenLabsOptions.cs` already exposes a `Model` property (today defaults to `eleven_turbo_v2`). Changes:

- Add canonical constants: `ElevenLabsModels.Flash25 = "eleven_flash_v2_5"`, `Turbo2 = "eleven_turbo_v2"`, `Multilingual2 = "eleven_multilingual_v2"`. Verify the official model id string against [ElevenLabs models docs](https://elevenlabs.io/docs/overview/models) at impl time.
- Update default to `Flash25` (additive — existing options accept any string; constant is documentation, default flip is documented as a "non-breaking default change" in CHANGELOG; callers who explicitly set `Model` see no change; callers using default see the new model).
- Add `OutputFormat` option (PCM 16-bit @ 16 kHz / 22.05 kHz / 24 kHz; today fixed via internal mapping). Required for Flash 2.5's lower-latency PCM frames.
- Add `LatencyOptimization` enum mapping to the provider's `optimize_streaming_latency` URL parameter (0-4 scale per their API).
- Update `ElevenLabsSpeechSynthesizer` WebSocket request to surface the new model + format + latency optimization.

**Backward compat:** existing options default behavior preserved when `Model` is explicitly set; new code paths only trigger when Flash 2.5 selected.

### Track 1.B — Deepgram Aura 2 TTS (new provider, **WebSocket** endpoint)

New folder `Asterisk.Sdk.VoiceAi.Tts/Deepgram/`:

- `DeepgramTtsOptions.cs` — ApiKey, Model (default `aura-2-thalia-en`; constants for available voices including the 2026 multilingual additions: Dutch, French, German, Italian, Japanese, Spanish), SampleRate (24000 default; 8000 / 16000 / 24000 / 48000 supported), Encoding (`linear16`, `mulaw`, `alaw`), Container (`none` for PCM).
- `DeepgramSpeechSynthesizer.cs` — **WebSocket** streaming via `wss://api.deepgram.com/v1/speak?model=<model>&encoding=<enc>&sample_rate=<sr>`. Auth header `Authorization: Token <key>`. Sends JSON `{ "type": "Speak", "text": "..." }` for each token chunk; receives binary PCM frames + JSON status messages. Implements `SpeechSynthesizer` base. **REST fallback NOT implemented in v1.15.3** — WS is the strategic surface; can be added if a customer reports WS unreachable.
- `DeepgramVoices.cs` — voice catalog constants generated from Deepgram's published voice list (Aura 2 voices only; verify catalog at impl time).
- `DependencyInjection/DeepgramTtsServiceCollectionExtensions.cs` — `AddDeepgramSpeechSynthesizer(opts => ...)` + auto-registered `TtsHealthCheck` that pings the Deepgram TTS lightweight endpoint on startup; degrades if 401/503.
- JSON serializer context entries in `Internal/VoiceAiTtsJsonContext.cs` (or new context if pattern dictates — match existing Deepgram STT pattern in `src/Asterisk.Sdk.VoiceAi.Stt/Deepgram/`).

Mirror Deepgram STT structure (already in repo as Nova-3 STT) for code-shape consistency. Same `HttpClient` setup pattern, same auth header, same WebSocket exception handling.

### Track 1.C — LMNT TTS (new provider, replacing Whisper)

New folder `Asterisk.Sdk.VoiceAi.Tts/Lmnt/`:

- `LmntTtsOptions.cs` — ApiKey, Voice (default an LMNT-curated low-latency voice id; constants for the most relevant voices), Model (e.g., `aurora`, `blizzard` — verify catalog at impl time), Format (PCM 16-bit), SampleRate (matches telephony — 8000/16000/24000), Speed (1.0 default).
- `LmntSpeechSynthesizer.cs` — chooses between LMNT's WebSocket streaming endpoint (preferred for sub-200 ms TTFA) and HTTP streaming endpoint (fallback for environments that block outbound WS). Implements `SpeechSynthesizer` base. Auth via `Authorization: <key>` header (verify exact header name at impl time).
- `LmntVoices.cs` — voice catalog constants curated for telephony.
- `DependencyInjection/LmntTtsServiceCollectionExtensions.cs` — `AddLmntSpeechSynthesizer(opts => ...)` + auto-registered `TtsHealthCheck`.
- JSON serializer context entries in `Internal/VoiceAiTtsJsonContext.cs`.

**Latency target:** TTFA < 200 ms in IT against the live LMNT API (env-tolerant ceiling, marketing claim is sub-200 ms).

### Track 2 — TTFA observability gap closure

Updates to [`src/Asterisk.Sdk.VoiceAi/Diagnostics/SpeechSynthesisMetrics.cs`](../../src/Asterisk.Sdk.VoiceAi/Diagnostics/SpeechSynthesisMetrics.cs):

```csharp
public static readonly Histogram<double> SynthesisTtfaMs =
    Meter.CreateHistogram<double>("tts.synthesis.ttfa_ms", "ms",
        "Time-to-first-audio: elapsed from synthesis start until first audio frame yielded to caller.");
```

Recording: `SpeechSynthesizer` base class wraps `SynthesizeAsync` → measures `Stopwatch` from method entry to first yielded `ReadOnlyMemory<byte>` (or framed equivalent). Tags: `voiceai.provider`, `tts.model`. Existing `tts.synthesis.latency_ms` (total) preserved unchanged.

All existing TTS providers (Azure, Cartesia, ElevenLabs, Speechmatics) inherit the new metric automatically — no per-provider changes if they all yield the first frame promptly. Spot-verify each provider's `SynthesizeAsync` does not buffer pre-yield (e.g., does not collect the full clip before returning the first byte). Add a unit test per provider asserting first-yield-before-EOF.

Documentation update: `src/Asterisk.Sdk.VoiceAi.Tts/README.md` lists the new histogram with default histogram buckets per Asterisk OTel conventions.

### Track 3 — Pro consumer wiring (Pro side, deferred to Pro v1.13.x-pro patch)

Out of R1.5 SDK scope per "Out of scope" above, but documented here so Pro brainstorm has the bridge ready:

- `Asterisk.Sdk.Pro.AgentAssist.AgentAssistBuilder.WithDeepgramTts(Action<DeepgramTtsOptions>?)` — fluent builder method.
- `WithLmntTts(Action<LmntTtsOptions>?)` — same.
- ElevenLabs Flash 2.5 is reachable today via `WithElevenLabsTts(opts => opts.Model = ElevenLabsModels.Flash25)`.

**Pro Track 3** lands in next Pro release alongside whichever R5 bump consumes SDK v1.15.3.

## Test strategy

- **Unit tests:** mock-based against fakes (`FakeSpeechSynthesizer` extended for new metrics). New ~25-35 tests across 3 providers + TTFA metric coverage.
- **Integration tests** (`RUN_LIVE_TESTS=true` env, gated):
  - ElevenLabs Flash 2.5: API key required, generates 5-second sample, asserts TTFA < 200 ms (env-tolerant ceiling above the 150 ms claim).
  - Deepgram Aura 2 WS: API key required, asserts TTFA < 250 ms.
  - LMNT WS: API key required, asserts TTFA < 250 ms.
- **AOT publish smoke:** run `tools/verify-aot.sh` (existing AOT validation script, also used by the `aot-check` CI job in `.github/workflows/ci.yml`). Must complete with **zero IL3050/IL3053**. Since v2 scope drops Whisper.net, no native interop deps are added; this should pass without new suppressions.
- **Latency benchmark:** `Tests/Asterisk.Sdk.Benchmarks/` BenchmarkDotNet smoke job — 10-run average TTFA per provider, gated by env var. Output appended to PR description for review.

## Risks / mitigations

| Risk | Likelihood | Mitigation |
|---|---|---|
| Deepgram Aura 2 WebSocket frame protocol differs from documented schema | Low-Medium | Q1 probe (½ day) before committing the synthesizer impl. |
| LMNT free-tier rate-limited, IT cannot run reliably | Medium | Mark IT `Trait("Category", "Live"); Trait("Provider", "LMNT")`. Document in CONTRIBUTING that LMNT live tests need a paid key (or skip). |
| Flash 2.5 TTFA claim unverifiable in test env | Low | TTFA histogram exists; env-tolerant ceiling in IT (200 ms vs 150 ms target). Marketing claim references provider documentation, not our test. |
| `tts.synthesis.ttfa_ms` recorded only when first frame yielded — providers that buffer pre-yield silently skew metric | Medium | Per-provider unit test asserts first byte yielded before total synthesis completes (i.e., the synthesizer is genuinely streaming, not collecting). Catches regressions in Azure/Cartesia/Speechmatics existing impls. |
| Deepgram REST fallback requested by a customer post-ship | Low | WS is sufficient for shipped use cases. Add REST as a follow-up patch if a customer reports WS unreachable in their network. |
| LMNT API auth header name / contract changes between spec and impl | Low | Verify at impl time against [LMNT API docs](https://docs.lmnt.com); pin tested API version in `LmntTtsOptions.ApiVersion`. |
